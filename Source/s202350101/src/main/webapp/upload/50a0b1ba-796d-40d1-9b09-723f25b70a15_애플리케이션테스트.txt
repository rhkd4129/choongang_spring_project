유효한 범위 (Valid Range):
	이상적인 경우, 입력 데이터 X는 0과 100 사이의 값이어야함

하한 경계 값 (Lower Boundary Value):
	 입력 데이터 X가 0인 경우를 고려

상한 경계 값 (Upper Boundary Value):
	이 경우, 입력 데이터 X가 100인 경우를 고려


-------------------------------------------------------------------------------------------
최소 경계값 (Minimum Boundary Value):
	최소 경계값은 입력 데이터 X가 0인 경우

내부 값 (Inner Value):
	내부 값은 주어진 범위 내에서 임의의 중간 값
	이상적인 경우 X 값은 1 또는 50과 같이 범위 내의 임의의 값이 될 수 있다.


최대 경계값 (Maximum Boundary Value):
	최대 경계값은 입력 데이터 X가 100인 경우를 나타냄
	이상적인 경우 X 값은 100이어야 함


-------------------------------------------------------------------------------------------
조건 (Condition): 조건은 IF 문에서 평가되는 부울 표현식 또는 논리 조건을 나타냄. 예를 들어, "age > 18" 및 "length_id > 16"은 두 개의 조건

결정 (Decision): 결정은 조건들의 조합에 따라 프로그램의 흐름을 결정하는 부분을 나타냄
 예로  "age > 18 or length_id > 16"은 하나의 결정


진리표 
조건1 (age > 18)		조건2 (length_id > 16)		결정 (A Logic 수행)

1				1			? (예를 들어, 1)
1				0			? (예를 들어, 0)
0				1			? (예를 들어, 1)
0				0			? (예를 들어, 0)

@BeforeClass 메서드: beforeClass() 메서드가 실행
이 메서드는 클래스 수준에서 실행되므로 가장 먼저 호출 출력은 1

@Before 메서드: before() 메서드가 실행
모든 테스트 메서드 실행 전에 호출. 출력은 3


@Test 메서드 (testCase1): testCase1() 메서드가 실행. 
이 메서드는 @Ignore 어노테이션이 있으므로 실행 x

@After 메서드: after() 메서드가 실행. 
모든 테스트 메서드 실행 후에 호출 출력은 4

@Test 메서드 (testCase2): testCase2() 메서드가 실행.
 @Ignore 어노테이션이 없으므로 실행 출력은 7
